Project Goal

NCR sought to create a new design system from the ground up. The goal of this project was to implement a design system that provided web & native app component designs, working code components, and the relevant documentation so that designers and developers at NCR could build products quickly and efficiently with a common design language.

Deciding Which Components to Include

My team quickly realized the importance of understanding and prioritizing which components would be included in the initial release of the design system. We knew that we wanted to release an early alpha version of the design system so that we could go ahead and start collecting feedback on it, so we needed to be intentional about which components made the cut.

Market Research - Design System Sites

Fortunately, there are lots of design systems to check out online. I used this fantastic index of design systems by @alexpate on Github to find some of the best design system sites out there from which to draw inspiration. As a researched, I took screenshots and created another affinity map to break design systems sites into smaller parts as I started to notice patterns across the various design system sites.

As I continued to research, I was starting to see patterns and I began to have an idea of what I thought I wanted the structure of our design system site to look like. I create a sitemap to capture this so that I could focus purely on information and site structure. By this time, we had narrowed down our list of components we wanted to include in the alpha release of the design system, so I made sure that the site structure would be able to accommodate the components we had selected, while leaving room for future iterations of new components.
